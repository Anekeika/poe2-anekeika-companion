# PoE 2 Companion — MVP Design

- **Дата:** 2026-06-13
- **Статус:** согласован, готов к написанию плана (writing-plans)
- **Проект:** `poe2-anekeika-companion` (git-сабмодуль в `Games/`)
- **Целевое место спеки после scaffold'а:** `Games/poe2-anekeika-companion/docs/superpowers/specs/2026-06-13-companion-mvp-design.md` (сейчас лежит в `Machine/Outputs/`, чтобы не создавать файлы в пути, который станет отдельным репо)

---

## 1. Цель

Компаньон для Path of Exile 2 внутри Obsidian + Claude. Агенты получают доступ к **актуальным данным персонажа и торговли**, знают **что метово** и **как крафтить**, и **подсказывают, какую вещь стоит заменить в билде**, понимая как строится DPS и выживание.

Репозиторий — отдельный git-сабмодуль, чтобы **делиться с друзьями**.

## 2. Scope MVP

1. **Дашборд персонажа** (3 блока): статы/экип/резисты персонажа (из PoB) + мета (что топ, где мой билд относительно меты) + курсы валют (экономика).
2. **Скрипты-сборщики данных** (без LLM): персонаж, экономика, мета, trade2. Складывают в локальный store.
3. **Агент-советчик** (единственный LLM-агент): подсказывает замену предмета, объясняет влияние на DPS/выживание. Опирается на реальный расчёт PoB2, а не на «на глаз».

### Non-goals (MVP)
- Не пишем свой движок расчёта урона — используем PoB2.
- Не строим автоматический крафт-планировщик (мета/крафт-знание — read-only справка через poe2db, не оптимизатор).
- Не делаем веб-сервис/мультиюзер-бэкенд. Всё локально, per-user.
- Не автоматизируем покупки/обмен на торговле (только чтение).

## 3. Проверенная feasibility источников (зафиксировано, чтобы не перепроверять)

| Что | Реально | Как / нюанс |
|---|---|---|
| Персонаж (шмот, скиллы, дерево, самоцветы) | ✅ чисто | Офиц. API `GET /character/poe2/<name>`, скоуп `account:characters`. OAuth (PKCE) один раз → refresh-токен. Realm = `poe2` |
| Профиль | ✅ | `GET /profile`, скоуп `account:profile` |
| Курсы валют / экономика | ✅ чисто | Офиц. `currency-exchange`, скоуп `service:cxapi`. Готовый JSON, нормализация к Chaos/Divine/Exalted |
| Мета — что топ | ✅ | poe.ninja PoE2 (builds + economy). Доступно без авторизации |
| База предметов / моды / крафт | ✅ | poe2db.tw, RePoE (mod-тиры, base stats) |
| Движок DPS/выживание (PoB2) | ✅ под PoE2 | `PathOfBuilding-PoE2` форк, `src/HeadlessWrapper.lua` (ветка `dev`): грузит билд, считает без GUI |
| Поиск конкретных шмоток (trade2) | ⚠️ грязно | Есть, но session-based + жёсткие рейт-лимиты. Изолируем |

**Главный барьер:** OAuth требует регистрации dev-приложения у GGG (`client_id`), возможно с заявкой/одобрением. Решение по ingestion — свой OAuth-клиент (выбрано).

**Важная находка по мостам (проверено):**
- `pob-mcp` (ianderse) — **только PoB1**, прибит к `ianderse/PathOfBuilding @ api-stdio`. К PoE2 не цепляется. **Не используем.**
- `sergeyklay/poe2-mcp-server` — **PoE2-нативный** (TypeScript/Node): poe.ninja + poe2scout + poe2db + wiki + RePoE. PoB только **декодит код, не считает**. Нет OAuth. MIT, **заархивирован 22.04.2026**. → форкаем как базу для meta/economy/item-db, владеем сами.
- Готового MCP-моста к расчётному движку PoB2 **нет** — пишем сами (см. §6).

## 4. Архитектура

### Репозиторий
`Games/poe2-anekeika-companion/` — отдельный git-репо, подключён в vault как сабмодуль. `Games/` — категория (на будущее — компаньоны других игр).

**Самодостаточность (требование от «поделиться»):** свой `CLAUDE.md`, project-scoped `.claude/` (агент + `.mcp.json`), скрипты, кросс-платформенный `scripts/install.{ps1,sh}`. Не зависит от внутренностей vault'а Павла. Друг: клонирует → install → авторизуется → работает.

### Модули (чёткие границы, тестируются отдельно)

| Модуль | Назначение | LLM | Происхождение |
|---|---|---|---|
| `auth/` | OAuth-клиент GGG (PKCE, refresh-токен). Интерфейс `get_token()`. Авторизация раз | нет | **пишем сами** |
| `ingest/` | Сборщики: персонаж, экономика, мета, trade2 → store | нет | мета/эконом/itemdb — **форк sergeyklay**; персонаж/trade2 — **сами** |
| `store/` | Плоские файлы JSON+MD: `data/character/<name>/`, `data/economy/`, `data/meta/`. Source of truth | нет | пишем сами |
| `pob/` | Форк `PathOfBuilding-PoE2` (вложенный сабмодуль) + **наш stdio-мост** поверх `HeadlessWrapper`. MCP-инструменты `load_build` / `get_stats` / `eval_swap` | мост | движок — форк PoB2; **мост пишем сами** |
| `dashboard/` | Рендер из store в Obsidian-заметку (3 блока). Не ходит в API | нет | пишем сами |
| `build-advisor` | **Единственный LLM-агент.** store + `pob` MCP + мета → советует замену, объясняет DPS/выживание | **да** | пишем сами (project agent) |

### Принцип: данные тащат скрипты, LLM включается интерактивно
Фетч персонажа/экономики/меты/trade2 — это **скрипты** (`ingest/`) + slash-команда `/poe2-refresh`, не LLM-агент. Гонять подписку на фоновый фетч — расточительство. LLM-агент в MVP **один** — советчик.

## 5. Поток данных

```
GGG OAuth ──token──> ingest/fetch-character ──> store/character/<name>/latest.json
currency-exchange ──> ingest/fetch-economy   ──> store/economy/
poe.ninja ─────────> ingest/fetch-meta       ──> store/meta/
trade2 ────────────> ingest/fetch-trade      ──> store/trade/   [хрупкий, изолирован]

store/ ──────────────────────────> dashboard (Obsidian-заметка, 3 блока)
character.json ──конвертер──> PoB-код ──> pob/ headless ──> реальные DPS/EHP

build-advisor (LLM):
  читает store/ + мету
  → "а если сменить кольцо?"
  → мутирует PoB-код → pob/ eval_swap → числа до/после
  → сравнивает, объясняет пользователю
LLM НИКОГДА не считает урон сам — редактирует структуру билда (текст), числа спрашивает у PoB.
```

## 6. PoB2-мост (`pob/`) — ядро и главный риск

- База: форк `PathOfBuilding-PoE2` как вложенный сабмодуль. В нём `HeadlessWrapper.lua` уже умеет `loadBuildFromJSON` / `loadBuildFromXML` и считать статы.
- Пишем **тонкий stdio/JSON-RPC слой** поверх `HeadlessWrapper` (порт подхода ianderse PR #9505 на форк PoE2), операции: `load_build`, `get_stats`, `eval_swap` (what-if по замене предмета/нода).
- Обёртка как MCP-сервер `pob2`, подключается project-scoped через `.mcp.json` компаньона.
- Конвертер `character.json (офиц. API) → PoB`: если `loadBuildFromJSON` ест схему, близкую к официальной `.build` (GGG 0.5), конвертер почти бесплатен. Проверяется в спайке.
- Зависимости: `luajit` в PATH (или `POB_CMD`). install-скрипт компаньона проверяет.

**Это не «утонем»:** мост — ограниченная задача (stdio-харнесс + горстка операций поверх готового движка). Прецедент ianderse на PoB1 доказывает паттерн; структура модулей в форке PoE2 та же.

## 7. Спайк №1 — гейт всего проекта

> Клонировать `PathOfBuilding-PoE2` → запустить `HeadlessWrapper.lua` через luajit → скормить реального персонажа (JSON официального API через `loadBuildFromJSON`) → получить вменяемые DPS/EHP.

- Зелёный → мозг под PoE2 рабочий, остальное плотницкая работа.
- Бонус-проверка: насколько `loadBuildFromJSON` совместим с официальной схемой (дешевизна конвертера).
- **Пока спайк не зелёный — за дашборд и советчик не беремся.**

## 8. Обработка отказов / деградация

- Токен протух → refresh. Рейт-лимит → backoff + отдаём из store (кэш).
- **trade2 изолирован** — самый хрупкий (session, лимиты). Его падение НЕ ломает дашборд и советчик (они на store). Держим тонким read-only «найди апгрейд для слота X».
- PoB headless упал → советчик честно: «точные числа недоступны», не выдаёт выдумку за факт.
- Нет свежих данных персонажа → дашборд показывает последний снапшот с пометкой времени.

## 9. Приватность и шеринг (критично — репо публичный)

Сабмодуль = отдельный repo, `.gitignore` vault'а на него **не распространяется**. Собственный `.gitignore` компаньона ОБЯЗАН прятать:
- `.env` / OAuth refresh-токен / любые секреты;
- `data/` — персонаж, токены, личные снапшоты;
- сгенерированный дашборд с личными данными.

Друзья получают **тулинг**, генерят свои данные локально. В git компаньона — только код/агенты/скрипты/шаблоны.

## 10. Интеграция с vault (не ломает command-center) — проверено

- `bootstrap.ps1` сканирует только `Machine Spirits/*` (стр. 179), **не** top-level → `Games/` не подхватывается как spirit. ✅
- Конфиг компаньона **project-scoped** (`Games/poe2-anekeika-companion/.claude/`), **не** ставится в глобальный `~/.claude` → нет клеша со spirit-агентами Павла. Vault-bootstrap не трогаем. ✅
- Vault трекает только gitlink + `.gitmodules` (указатель на коммит) → нет churn'а от игровых данных в истории vault'а. ✅
- PoB2 и форк sergeyklay — вложенные сабмодули компаньона. Жирный PoB2 помечаем `update = none` в `.gitmodules` компаньона, чтобы `git submodule update --recursive` в vault-bootstrap не тянул его без спроса (тянется явным install'ом). ⚠️ заложить при scaffold'е.
- Obsidian: новая папка с заметками не ломает индексацию. JSON-данные при желании исключить из Obsidian. ✅

## 11. Кросс-платформенность (правило vault'а)

- Скрипты: PowerShell Core (`pwsh`) **и** bash/POSIX, либо Node-runner. PS-файлы — UTF-8 with BOM (PS 5.1).
- Пути — через переменные/конфиг, не hardcoded.
- Junction → symlink (`New-Item -ItemType SymbolicLink` / `ln -s`).
- Секреты через `env`.
- `install.{ps1,sh}` — обе ОС.

## 12. Дашборд — подход

**Markdown-first**: генерим Obsidian-заметку (таблицы статов/экип/резистов из PoB + блок меты + блок курсов валют). Нативно, без плагинов. HTML/Canvas-апгрейд — потом. Конкретные макеты — визуально на этапе реализации.

## 13. Что форкаем vs пишем сами

| Форкаем (владеем) | Пишем сами |
|---|---|
| `PathOfBuilding-PoE2` (движок) | OAuth-клиент (`auth/`) |
| `sergeyklay/poe2-mcp-server` (мета/эконом/poe2db) | stdio-мост к PoB2 (`pob/`) |
| | fetch персонажа + trade2 (`ingest/`) |
| | store, дашборд, агент-советчик |

## 14. Открытые вопросы / риски

1. **Спайк №1** (см. §7) — зрелость headless PoE2 + совместимость `loadBuildFromJSON` с официальной схемой. Гейт.
2. **GGG dev-приложение** — сроки/условия одобрения OAuth-клиента. Фолбэк на ранний этап: ручной импорт PoB-кода, пока ждём.
3. **trade2** — реальные рейт-лимиты и потребность в session-cookie; насколько стабильно. Держим изолированным.
4. **Лицензии форков** — PoB (MIT), sergeyklay (MIT). Проверить совместимость при публикации компаньона.
5. **Версионность PoE2 EA** — игра в раннем доступе, схемы/моды меняются. poe2db/RePoE и форк PoB2 надо уметь обновлять.

## 15. Ориентировочная последовательность (детали — в плане)

0. Scaffold репо компаньона (структура, `.gitignore`, project `.claude/`, install-скрипты) + перенос спеки внутрь.
1. **Спайк №1** — PoB2 headless считает реального персонажа.
2. `auth/` OAuth + `ingest/fetch-character` → store.
3. `pob/` stdio-мост + MCP `pob2`.
4. Дашборд (блок персонажа) на реальных данных.
5. Форк sergeyklay → `ingest` меты/экономики → блоки меты и курсов в дашборде.
6. `build-advisor` агент (store + pob + мета).
7. trade2 (изолированный модуль) + интеграция в советы.
